# Screenshots

Deterministic UI captures generated by the Playwright screenshot pipeline from
the **seeded demo dataset** — these are real application states, not mockups.

| File | State |
|------|-------|
| `01-dashboard-overview.png` | Dashboard: KPI strip, routing donut, throughput trend |
| `02-run-detail-trace.png`   | Run detail / trace viewer for an auto-saved run |
| `03-review-edit-state.png`  | A flagged run with the "edit output before approving" editor open |
| `04-post-approval.png`      | The review panel after a decision is recorded |

## Regenerate

From `caretrace/frontend` (Playwright starts an isolated, freshly-seeded backend
and a production frontend build automatically — nothing to start by hand):

```bash
npm run e2e:screens
```

Images are written back into this folder (`docs/screenshots/`). Refresh them
whenever the relevant UI changes, and commit the updated PNGs.

> The captures use a production build, so the dev-only debug overlay is absent —
> they match what a real user sees. See `docs/DEMO_RUNBOOK.md` → "End-to-end
> tests & screenshots" for prerequisites and how the environment is orchestrated.
